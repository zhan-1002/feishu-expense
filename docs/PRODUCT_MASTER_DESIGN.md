# 商品主表与全链路架构设计

## 1. 背景

选品任务流程是飞书多维表格应用的核心。产品生命周期从"被选中可进入采购"开始，经历采购、仓储、销售等环节，各环节产生的人员开销和支出都密切相关。

为支持各部门独立维护各自工作环节，同时能在总表看清数据脉络，设计基于飞书**父子记录**的核心架构。

---

## 2. 核心总表

### 2.1 商品主表（PRODUCT）

作为所有业务子记录的父表锚点。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商品ID | 自动编号 | 主键 |
| 商品标题 | 文本 | |
| 商品链接 | 超链接 | |
| 商品图片 | 附件 | |
| 生命周期状态 | 单选 | 选品中/采购中/入库中/在售/已下架 |
| 创建时间 | 创建时间 | |
| 最后更新 | 修改时间 | |

**设计说明**：
- 不设"当前负责人"字段，因子记录展开即可看到各环节负责人
- 生命周期状态仅标记当前所处大阶段

### 2.2 人员表（STAFF）

作为各表中人员字段的统一来源。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 员工ID | 自动编号 | 主键 |
| 姓名 | 文本 | |
| 飞书账号 | 人员 | 关联飞书用户 |
| 部门 | 单选 | 供应链/采购/销售/仓储/财务 |
| 角色 | 单选 | 专员/主管/管理员 |
| 状态 | 单选 | 在职/离职 |
| 入职时间 | 日期 | |
| 备注 | 多行文本 | |

## 3. 子表扩展方向

以下子表通过"关联记录"字段关联商品主表，在父记录中可展开查看。

### 3.1 选品记录表（现有 selection-erp 扩展）

- 关联商品主表
- 选品状态、负责人、结论等
- 现有业务表、任务表、通知表由供应链部门自维护

### 3.2 采购记录表（待建）

- 关联商品主表
- 采购状态、采购员、供应商、成本、数量、入库时间等

### 3.3 SKU表（待建）

- 关联商品主表
- SKU编码、规格属性、成本价、售价、库存等

### 3.4 销售记录表（待建）

- 关联SKU表（二级关联 → 商品主表）
- 店铺、负责人、销量、销售额等

### 3.5 其他子表（按需扩展）

- 仓储记录表
- 售后记录表
- 费用明细表
- ...

## 4. 表间关系

```
┌─────────────────┐
│    商品主表      │ ← 所有业务子表的父记录
│   (PRODUCT)     │
└─────────────────┘
        ↑
        │ (关联记录 - 父子记录)
        │
   ┌────┴────┬─────────┬─────────┐
   │         │         │         │
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│选品表│ │采购表│ │SKU表│ │...   │
└──────┘ └──────┘ └──────┘ └──────┘
                        │
                        │ (二级关联)
                        ↓
                  ┌──────────┐
                  │ 销售表   │
                  └──────────┘


┌─────────────────┐
│    人员表       │ ← 各表人员字段来源
│   (STAFF)       │
└─────────────────┘
        ↑
        │ (关联)
        │
   各业务表负责人字段
```

## 7. 设计原则

1. **总表极简**：只存核心标识和当前状态，不存汇总数据
2. **子表自治**：各部门独立维护各自子表
3. **父子关联**：通过飞书关联记录实现数据脉络可视化
4. **按需扩展**：子表根据业务发展逐步增加，不预先过度设计
